# Свежов — Реферальная система

WPF-приложение (настольное приложение) для управления базой клиентов и реферальной бонусной программой сети «Свежов». Позволяет регистрировать пользователей, начислять приветственные баллы и отслеживать реферальные связи (кто кого пригласил).

## 🚀 Основные возможности

1. **Управление пользователями**:
   - Просмотр общего списка клиентов (ФИО, номер телефона, реферальный код, баланс баллов).
   - Поиск клиентов по имени или номеру телефона.
   - Добавление новых пользователей с автоматической генерацией уникального 8-значного реферального кода.
   - Удаление пользователей из базы данных.

2. **Реферальная программа**:
   - Указание реферального кода пригласившего при регистрации нового пользователя.
   - Привязка рефералов 1-го уровня к пригласившему пользователю.
   - Просмотр списка всех приглашенных пользователей (рефералов) для каждого клиента.

3. **Бонусная система и транзакции**:
   - Автоматическое начисление **50 приветственных баллов** каждому новому пользователю.
   - Автоматическое начисление **100 бонусных баллов** пригласившему пользователю при успешной регистрации его реферала.
   - Ведение и просмотр детальной истории транзакций баллов для каждого клиента (тип операции, количество баллов, описание, дата и время).

---

## 🛠 Технологический стек

- **Платформа**: .NET / WPF (Windows Presentation Foundation)
- **Архитектура**: MVVM (Model-View-ViewModel) с использованием паттерна репозитория
- **База данных**: SQLite (локальный файл `svezhov.db`)
- **ORM**: Dapper (для удобного и быстрого доступа к данным)

---

## 📂 Структура базы данных

База данных содержит две основные таблицы:

### 1. `Users` (Пользователи)
- `ID` — уникальный идентификатор пользователя (Primary Key, Autoincrement)
- `FullName` — ФИО клиента
- `PhoneNumber` — контактный телефон
- `ReferralCode` — уникальный реферальный код пользователя
- `InviterID` — ID пригласившего пользователя (Foreign Key, указывает на `Users.ID`)
- `PointsBalance` — баланс бонусных баллов
- `RegistrationDate` — дата и время регистрации

### 2. `Transactions` (История транзакций)
- `ID` — уникальный идентификатор транзакции (Primary Key, Autoincrement)
- `UserID` — ID пользователя, к которому относится операция (Foreign Key)
- `OperationType` — тип операции (например, «Начисление»)
- `PointsAmount` — количество начисленных или списанных баллов
- `Description` — описание операции (например, «Приветственный бонус»)
- `TransactionDate` — дата и время совершения транзакции

## 💻 Инструкция по запуску

1. Убедитесь, что на компьютере установлен **.NET SDK** (версия 6.0 или новее).
2. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/baxily/MDK011_Project.git
   ```
3. Откройте папку проекта в среде разработки (например, Visual Studio или Rider).
4. Запустите проект. При первом старте программа автоматически создаст базу данных `svezhov.db` в корневой папке и наполнит её тестовыми данными (50 случайных пользователей с реферальными связями и историей транзакций).
